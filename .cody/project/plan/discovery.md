# Discovery
这份文档记录了项目最初的想法和迭代的探索过程。

## Starting Prompt
我需要一个竞品能力对比可视化工具，让你在和研发、Sales 讨论时能快速、直观地呈现 Omada 与竞品的能力差异。

### 核心场景
- **内部评审会议**：与研发团队讨论产品能力差距，确定优化优先级
- **销售支持**：给 Sales 提供可视化的竞品对比材料，辅助客户沟通
- **战略规划**：多维度评估产品竞争力，支撑产品路线图决策
- **动态更新**：竞品能力变化时，快速更新对比数据

### 用户角色
- **创建者**（PM团队）：创建、编辑、维护对比数据
- **查看者**（研发/Sales）：查看对比结果，理解能力差异

### 初始功能需求
1. 图表展示区（顶部）
   - Tab 导航，支持多雷达图切换
   - 主雷达图显示一级维度对比
   - 子维度雷达图级联展开
   - 图表交互（系列显隐、全屏、导出）

2. 设置区（下方）
   - 雷达图管理（增删改查、复制）
   - 系列（Vendor）管理（增删、颜色、标记、排序）
   - 维度管理（增删、名称、说明、权重、分数、排序）
   - 子维度管理（展开收起、自动计算）
   - 总分计算与排名

3. 数据导入/导出
   - 支持 Excel、JSON、图片、PDF 格式

4. 数据持久化
   - 本地存储（LocalStorage/IndexedDB）
   - 可选云存储扩展

### 技术栈建议
- 框架：React + TypeScript
- 图表：ECharts
- UI：Ant Design 5 或 Shadcn/ui + Tailwind
- 工具：Vite、SheetJS、dnd-kit

## Discovery Summary

### 项目愿景
构建一个**竞品能力对比可视化工具**，替代现有的 Excel 方式，为 Omada 团队提供一个直观、易用、可协作的竞品分析平台。

### 目标用户
- **PM 团队**：创建和维护竞品对比数据
- **研发团队**：查看能力差距，确定优化优先级
- **Sales 团队**：获取可视化对比材料，辅助客户沟通

### 核心问题
当前使用 Excel 做竞品对比存在两大痛点：
1. **不直观**：表格形式难以快速理解多维度能力差异
2. **不方便协作**：Excel 文件传来传去，难以实时同步更新

### 期望结果
团队成员觉得工具"好用"，能够：
- 在会议中快速、直观地展示竞品差异
- 方便地协作更新和维护数据
- 动态跟踪竞品能力变化

### 核心功能（v1.0）
1. **多雷达图管理**：Tab 切换、拖拽排序、增删改
2. **可视化对比**：主雷达图 + 子维度级联雷达图
3. **数据管理**：
   - 雷达图：通常 2-5 个
   - 一级维度：4-8 个
   - 子维度：5-6 个
   - 竞品系列：2-3 个（含自身）
4. **评分系统**：
   - 0-10 分整数评分
   - 维度权重强制 100%
   - 子维度自动按权重计算父维度分数
5. **总分排名**：实时计算和展示
6. **导入导出**：Excel、JSON、图片（PNG/SVG）
7. **本地存储**：浏览器自动保存
8. **轻量级分享**：
   - 生成分享链接（只读/可编辑）
   - 使用 Vercel KV 或 URL 编码
   - 无需复杂后端和用户系统
9. **中英双语**：界面文案支持切换

### 技术约束
- **前端为主**：React + TypeScript + Vite
- **轻量级后端**：仅用于分享功能，考虑 Vercel KV 或无后端方案
- **无复杂数据库**：不引入 Supabase 等重型方案
- **部署平台**：Vercel

### 设计要求
- **苹果风格**：简洁、现代、留白、优雅
- **桌面优先**：会议投屏和个人电脑使用
- **移动适配**：仅查看，不编辑
- **主题切换**：支持深色/浅色模式

### 成功指标
团队成员主动使用该工具，替代 Excel 成为竞品对比的首选方案。
