# Product Requirements Document (PRD)
这份文档定义了产品的"是什么"和"为什么"。

## Section Explanations
| Section           | Overview |
|-------------------|--------------------------|
| Summary           | 设定产品的高层次上下文 |
| Goals             | 阐述产品的目的——核心的"为什么" |
| Target Users      | 明确受众，对塑造功能和优先级至关重要 |
| Key Features      | 描述为实现目标需要构建什么——"是什么"的一部分 |
| Success Criteria  | 定义什么样的结果能验证目标 |
| Out of Scope      | 防止范围蔓延并设定边界 |
| User Stories      | 高层次的故事保持对用户需求的关注（为什么）并指导构建什么 |
| Assumptions       | 明确上下文和未知因素——对产品清晰度至关重要 |
| Dependencies      | 识别障碍和关键集成——对规划依赖关系和现实性有价值 |

## Summary

**Radar Bench** 是一个竞品能力对比可视化工具，通过交互式雷达图帮助 Omada 团队直观地展示与竞品的能力差异，支持多维度评估、实时协作和数据分享，旨在替代传统的 Excel 表格方式，提升内部评审、销售支持和战略规划的效率。

## Goals

1. **提升直观性**：通过可视化雷达图，让多维度能力对比一目了然，替代难以理解的 Excel 表格
2. **简化协作**：支持轻量级的分享和协作编辑，解决 Excel 文件传递和同步的痛点
3. **动态维护**：让竞品数据的更新和维护变得简单快捷
4. **提高效率**：在会议演示、销售沟通、战略规划等场景中节省时间，快速传达关键信息
5. **易用性优先**：打造团队"觉得好用"的工具，降低学习成本

## Target Users

### 主要用户角色

1. **PM 团队（创建者）**
   - 创建竞品对比数据
   - 维护和更新评估维度
   - 分享对比结果给其他团队
   - 主要使用场景：战略规划、产品路线图决策

2. **研发团队（查看者/协作者）**
   - 查看产品能力差距
   - 协作补充技术维度的评分数据
   - 主要使用场景：内部评审会议、技术优先级讨论

3. **Sales 团队（查看者）**
   - 查看可视化的竞品对比材料
   - 在客户沟通中展示产品优势
   - 主要使用场景：销售演示、客户答疑

### 使用环境
- **主要设备**：桌面电脑（会议室投屏、个人办公）
- **辅助设备**：移动设备（仅查看，不编辑）

## Key Features

### 1. 多雷达图管理
- 支持创建多个对比场景（通过 Tab 切换）
- Tab 可拖拽排序、重命名、复制、删除
- 预期数量：2-5 个雷达图

### 2. 可视化对比
- **主雷达图**：展示一级维度（4-8 个）的能力对比
- **子维度雷达图**：点击一级维度后级联展开，显示子维度（5-6 个）详情
- 多竞品系列叠加显示（通常 2-3 个，含自身）
- 图表交互：系列显隐、悬浮提示、全屏查看

### 3. 评分系统
- **评分范围**：0-10 分（整数）
- **权重管理**：维度权重强制总和为 100%
- **自动计算**：子维度分数按权重自动计算父维度分数
- **总分排名**：实时计算每个竞品的总分并排名

### 4. 数据管理
- **维度管理**：增删改维度、设置名称/说明/权重、拖拽排序
- **子维度管理**：展开/收起、增删改、自动计算
- **竞品管理**：增删竞品系列、自定义名称/颜色/标记类型、拖拽排序

### 5. 导入导出
- **导出格式**：Excel (.xlsx)、JSON、图片 (PNG/SVG)
- **导入格式**：Excel (.xlsx)、JSON
- **模板下载**：提供标准 Excel 导入模板
- **数据校验**：导入时校验格式并提示错误

### 6. 本地存储
- 浏览器自动保存（LocalStorage/IndexedDB）
- 防止数据丢失
- 支持多份数据方案管理

### 7. 轻量级分享与协作
- **分享链接生成**：支持"仅查看"和"可编辑"两种模式
- **轻量级后端**：使用 Vercel KV 或 URL 编码方案
- **无需登录**：无复杂用户权限系统
- **协作场景**：
  - 分享给 Sales 查看（只读）
  - 分享给研发协作编辑（可编辑）

### 8. 国际化支持
- 中英文界面切换
- 仅界面文案翻译，用户输入内容不翻译

### 9. 设计风格
- **视觉风格**：苹果风格（简洁、现代、留白、优雅）
- **主题切换**：支持深色/浅色模式
- **响应式设计**：桌面端优先，移动端适配查看

## Success Criteria

1. **采用率**：团队成员主动选择使用该工具，而不是 Excel
2. **易用性**：新用户无需培训即可快速上手（< 5 分钟）
3. **使用频率**：在内部会议和 Sales 演示中被定期使用
4. **正面反馈**：团队成员普遍反馈"好用"、"直观"、"方便"
5. **协作效率**：数据更新和同步比 Excel 方式更快捷

## Out of Scope

### v1.0 不包含的功能

1. **移动端编辑**：移动设备仅支持查看，不支持编辑操作
2. **复杂权限系统**：无用户登录、角色管理、细粒度权限控制
3. **用户账号系统**：无需注册、登录、密码找回等功能
4. **复杂后端数据库**：不引入 Supabase、PostgreSQL 等重型数据库方案
5. **内容自动翻译**：国际化仅覆盖界面文案，不翻译用户输入内容
6. **PDF 报告导出**：优先级较低，可后续版本考虑
7. **历史版本管理**：不支持数据版本历史和回滚
8. **实时协作编辑**：无 Google Docs 式的多人同时编辑能力
9. **评论功能**：无针对维度或分数的评论讨论功能
10. **AI 辅助分析**：无智能推荐、趋势预测等 AI 功能

## User Stories

### PM 团队
- 作为 PM，我希望能快速创建竞品对比雷达图，以便在战略规划会议中展示产品差距
- 作为 PM，我希望能轻松调整维度权重和评分，以便动态反映竞品变化
- 作为 PM，我希望能分享对比结果给团队，以便大家在不同场景下查看和使用

### 研发团队
- 作为研发人员，我希望能直观看到技术能力差距，以便确定优化优先级
- 作为研发人员，我希望能协作补充技术维度的评分，以便数据更准确全面
- 作为研发人员，我希望在会议中快速查看雷达图，以便理解产品定位

### Sales 团队
- 作为 Sales，我希望能获取可视化的竞品对比材料，以便在客户沟通中使用
- 作为 Sales，我希望能在移动设备上查看对比图，以便随时随地展示给客户
- 作为 Sales，我希望界面支持中英文切换，以便服务不同语言的客户

## Assumptions

1. **用户环境**：
   - 用户有现代浏览器（Chrome、Safari、Edge、Firefox）
   - 桌面端屏幕分辨率 ≥ 1280px
   - 网络连接稳定（用于分享功能）

2. **数据规模**：
   - 雷达图数量：2-5 个
   - 一级维度：4-8 个
   - 子维度：5-6 个/维度
   - 竞品系列：2-3 个
   - 数据规模适合前端渲染，无性能瓶颈

3. **使用频率**：
   - 数据更新频率不高（周/月级别）
   - 分享链接访问量不大（< 100 次/天）
   - 轻量级后端方案能满足需求

4. **技术能力**：
   - 用户无需技术背景即可使用
   - 数据导入时用户能按模板格式准备 Excel

5. **部署环境**：
   - 项目托管在 Vercel 平台
   - 使用 Vercel 提供的免费/付费服务

## Dependencies

### 技术依赖

1. **前端框架**：React + TypeScript + Vite
2. **图表库**：ECharts（雷达图、交互、导出）
3. **UI 组件库**：Shadcn/ui + Tailwind CSS（苹果风格）
4. **国际化**：react-i18next 或类似方案
5. **拖拽功能**：dnd-kit 或 react-beautiful-dnd
6. **Excel 处理**：SheetJS (xlsx)
7. **存储方案**：LocalStorage / IndexedDB (Dexie.js)

### 平台依赖

1. **部署平台**：Vercel（托管、CDN、Serverless Functions）
2. **分享后端**：Vercel KV (Redis) 或 Vercel Blob（轻量级存储）

### 外部依赖

1. **浏览器兼容性**：需支持现代浏览器特性（ES6+、LocalStorage、Canvas）
2. **无第三方 API**：不依赖外部数据源或服务

### 团队依赖

1. **内容输入**：需要 PM 团队提供初始的维度和评分数据
2. **测试反馈**：需要研发和 Sales 团队试用并提供反馈
